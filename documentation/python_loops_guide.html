The for Loop

A for loop repeats a block of code once for each item in a sequence — a list, string, or range of numbers. You always know what you're iterating over before the loop starts.

📌 Core idea
"Do this for every item in that collection." Python automatically moves to the next item and stops when it runs out.

Syntax

Python
for variable in sequence:
    # indented body runs once per item
    do_something(variable)

Example — iterating a list

Python
fruits = ['apple', 'banana', 'cherry']

for fruit in fruits:
    print(fruit)

# Output:
# apple
# banana
# cherry

Example — iterating a string

Python
for letter in "hello":
    print(letter)
# h  e  l  l  o  (one per line)

✅ Remember
The loop variable (e.g. fruit) is just a name you choose. Python assigns the next item to it automatically at the start of each iteration.

The range() Function

Use range() when you need to loop a specific number of times or generate a sequence of numbers.

range(stop)
Counts from 0 up to, but not including, stop.
range(5) → 0, 1, 2, 3, 4
range(start, stop)
Counts from start up to, but not including, stop.
range(1, 6) → 1, 2, 3, 4, 5
range(start, stop, step)
Counts by step each time.
range(0, 10, 2) → 0, 2, 4, 6, 8
range(stop, start, -1)
Counts down using a negative step.
range(5, 0, -1) → 5, 4, 3, 2, 1
Python — range() examples
for i in range(5):
    print(i)          # 0 1 2 3 4

for i in range(1, 6):
    print(i)          # 1 2 3 4 5

for i in range(0, 10, 2):
    print(i)          # 0 2 4 6 8
⚠️ Common mistake
range(5) gives 0 through 4, not 1 through 5. The stop value is never included. To loop 5 times printing 1–5, use range(1, 6).

The while Loop

A while loop keeps running as long as a condition is True. Use it when you don't know in advance how many iterations you need — you're waiting for something to change.

📌 Core idea
"Keep doing this until the condition becomes False." You are responsible for making sure that eventually happens.

Syntax

Python
while condition:
    # runs while True
    do_something()
    # update condition!

Countdown example

Python
count = 3

while count > 0:
    print(count)
    count -= 1

print("Done!")
# 3  2  1  Done!

Execution flow

While loop

User input example

Python
answer = ""

while answer != "quit":
    answer = input("> ")
    print("You typed:", answer)

print("Goodbye!")
🚫 Infinite loop danger
If the condition never becomes False, the loop runs forever and your program freezes. 
# DANGER — never stops:
while True:
    print("forever")

Loop Keywords: break and continue

Two special keywords give you extra control over loop flow.

break

Immediately exits the entire loop, no matter how many items remain.

for i in range(10):
    if i == 5:
        break      # stops at 5
    print(i)
# 0 1 2 3 4
continue

Skips the rest of the current iteration and jumps to the next one.

for i in range(6):
    if i == 3:
        continue   # skips 3
    print(i)
# 0 1 2 4 5

Useful Loop Patterns

Accumulator pattern

Build up a total by updating a variable inside the loop.

Python
total = 0
numbers = [5, 10, 15, 20]

for n in numbers:
    total += n

print(total)   # 50

enumerate() — index + value

When you need both the position and the value of each item.

Python
fruits = ['apple', 'banana', 'cherry']

for i, fruit in enumerate(fruits):
    print(i, fruit)
# 0 apple
# 1 banana
# 2 cherry

for vs. while — When to Use Which?

The choice usually comes down to one question: do you know what you're iterating over before the loop starts?

for loop while loop
Use when… You know what to iterate over (list, range, string) You're waiting for a condition to change
Iteration control Python handles it automatically You must update something in the body
Typical use cases Process every item in a list; repeat N times User input loops; game loops; wait for an event
Risk of infinite loop? No — stops when the sequence ends Yes — if condition never becomes False
Quick rule "I have a list / count" "I'm waiting until something happens"
💡 Default choice
When in doubt, reach for for. Most beginner problems give you a list or a number — for loops are safer, shorter, and harder to accidentally break.

Common Errors and Fixes

Error typeWhat it looks likeFix
IndentationError Loop body not indented (4 spaces required) Indent all lines inside the loop consistently
Infinite loop Program runs forever, output repeats endlessly Ensure the condition variable is updated inside the body
Off-by-one range(5) used when 1–5 was intended (gives 0–4) Use range(1, 6) to start at 1 and end at 5
Wrong variable Using list name instead of loop variable inside body Check you're using the loop variable (e.g. fruit, not fruits)
Modifying list Changing a list while looping over it — unexpected skips Iterate over a copy: for item in my_list.copy():

Nested Loops

A loop can be placed inside another loop. The inner loop runs completely for every single iteration of the outer loop.

Python
for row in range(1, 4):      # 1, 2, 3
    for col in range(1, 4):  # 1, 2, 3
        print(row, col)

# 1 1 / 1 2 / 1 3
# 2 1 / 2 2 / 2 3
# 3 1 / 3 2 / 3 3  →  9 lines total
⚠️ Watch the indentation
Each level of nesting adds another 4-space indent. The inner loop body is indented 8 spaces. Code after both loops (back at 0 spaces) runs only once when everything is done.
